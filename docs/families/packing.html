Packing & Cutting

Resource Allocation Under Capacity Constraints

Three foundational combinatorial optimization problems addressing how to best allocate items to limited-capacity containers. From the classical knapsack problem to industrial cutting stock optimization, this family covers exact dynamic programming, branch-and-bound methods, approximation algorithms with provable guarantees, and genetic algorithm metaheuristics.

KP01  |  max Σ vixi  s.t.  Σ wixi ≤ W,  xi ∈ {0,1}
NP-Hard (weakly) 37 Tests 5 Python Files 8 Test Classes

Given n items, each with a weight wi and value vi, and a knapsack with capacity W, select a subset of items to maximize total value without exceeding the weight limit. The binary constraint xi ∈ {0,1} means each item is either fully included or excluded. Weakly NP-hard: it admits a pseudo-polynomial time DP algorithm in O(nW) but no known polynomial-time exact method.

maximize Σi=1..n vi · xi subject to Σi=1..n wi · xi ≤ W // capacity constraint xi ∈ {0, 1} ∀ i = 1, ..., n // binary selection // LP relaxation upper bound: sort by vi/wi, fill greedily, split last item UB = Σi∈S vi + (W - Σi∈S wi) · (vk / wk)
Algorithms
Algorithm Type Complexity Guarantee Description
Dynamic Programming Exact O(n · W) Optimal Bottom-up tabulation. Pseudo-polynomial; practical for moderate W.
Branch & Bound Exact O(2n) Optimal LP relaxation upper bound, greedy warm-start. Prunes search tree effectively for sparse instances.
Greedy Value-Density Heuristic O(n log n) — Sort items by vi/wi descending, pack greedily. No constant-factor guarantee alone.
Greedy Max-Value Heuristic O(n log n) — Sort by value descending, pack greedily. Complements density-based approach on correlated instances.
Greedy Combined Heuristic O(n log n) 1/2-approx Best of density and max-value greedy. Guarantees at least half the optimal value.
Genetic Algorithm Metaheuristic O(G · P · n) — Binary encoding, uniform crossover, repair operator for infeasible solutions. Population-based search.

1D Bin Packing
BPP1D  |  min # bins  s.t.  Σi∈Bj si ≤ C  ∀ j
NP-Hard (strongly) 29 Tests 3 Python Files 7 Test Classes
Container Loading Memory Allocation VM Provisioning Truck Loading Tape Backup

Given n items with sizes si and identical bins of capacity C, pack all items into the minimum number of bins. Unlike knapsack, there is no selection: all items must be packed, and the objective is to minimize container usage. Strongly NP-hard, meaning no pseudo-polynomial algorithm exists unless P = NP. The problem admits excellent approximation algorithms, notably FFD with its (11/9)OPT + 6/9 guarantee.

minimize Σj=1..n yj // number of bins used subject to Σi=1..n si · xij ≤ C · yj ∀ j // bin capacity Σj=1..n xij = 1 ∀ i // each item packed once xij, yj ∈ {0, 1} // binary assignment // Lower bounds: L1 = ⌈ Σ si / C ⌉ // continuous relaxation L2 ≥ L1 // improved bound via item pairing
Algorithms
Algorithm Type Complexity Guarantee Description
First Fit (FF) Heuristic O(n2) 17/10 OPT + 7/10 Place each item in the first bin that has sufficient remaining capacity. Online algorithm.
First Fit Decreasing (FFD) Heuristic O(n log n) 11/9 OPT + 6/9 Sort items by size descending, then apply First Fit. Best-known offline approximation in practice.
Best Fit Decreasing (BFD) Heuristic O(n log n) 11/9 OPT + 6/9 Sort descending, place in bin with tightest remaining capacity. Same worst-case as FFD, sometimes better in practice.
Genetic Algorithm Metaheuristic O(G · P · n) — Permutation encoding with FF decoder. Evolves item orderings; FF decodes each chromosome into a valid packing.

1D Cutting Stock
CSP1D  |  min # rolls  s.t.  demand di satisfied  ∀ i
NP-Hard 21 Tests 2 Python Files 6 Test Classes
Paper Industry Steel Cutting Textile Manufacturing Glass Cutting Cable Production

Given stock material of length L and m item types with lengths li and demands di, cut stock rolls to satisfy all demands using the minimum number of rolls. This generalizes Bin Packing: identical items are interchangeable, enabling cutting patterns where multiple copies of the same item type are cut from a single roll. The classical column generation approach (Gilmore-Gomory, 1961) solves the LP relaxation by generating patterns on-the-fly.

minimize Σp=1..P yp // number of rolls used subject to Σp=1..P aip · yp ≥ di ∀ i // demand satisfaction Σi=1..m li · aip ≤ L ∀ p // pattern feasibility yp ∈ Z+ // integer roll count // aip = number of type-i items cut in pattern p // Reduces to Bin Packing when all di = 1
Algorithms
Algorithm Type Complexity Description
Greedy Largest-First Heuristic O(m · D) Fill each roll by repeatedly selecting the largest item with remaining demand that fits. Simple but effective for balanced instances.
FFD-Based Heuristic O(D log D) Expand demands into individual items, apply First Fit Decreasing, then aggregate cutting patterns. Leverages FFD's strong approximation guarantee.
